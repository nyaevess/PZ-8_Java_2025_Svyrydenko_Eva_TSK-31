**Відповіді на контрольні питання**

---

Чим інтерфейс Set відрізняється від інтерфейсів Collection та List?
Collection є базовим інтерфейсом для всіх колекцій. List дозволяє дублікати, зберігає порядок і підтримує доступ за індексом. Set не допускає дублікатів, не має доступу за індексом і зазвичай не гарантує порядок елементів.

HashSet відрізняється від TreeSet?
HashSet працює на основі хеш-таблиці, не зберігає порядок і має швидші операції. TreeSet базується на відсортованому дереві, зберігає елементи у відсортованому вигляді та працює повільніше.

Що таке хешування, хеш, хеш-код, хеш-функція?
Хешування - це процес перетворення об’єкта у числове значення. Хеш-функція — алгоритм обчислення цього значення. Хеш-код - результат роботи хеш-функції. Хеш використовується для швидкого доступу до об’єкта.

Яким вимогам має відповідати коректна хеш-функція?
Однакові об’єкти повинні мати однаковий хеш-код. Хеш-код повинен залишатися сталим, поки об’єкт не змінюється. Хеш-функція повинна бути узгодженою з equals().

Які властивості притаманні хорошій хеш-функції?
Рівномірний розподіл значень. Мінімальна кількість колізій. Швидке обчислення. Використання значущих полів об’єкта.

Якими критеріям мають відповідати об’єкти для того, щоб посилання на них можна було зберігати у HashSet?
Об’єкти повинні коректно перевизначати equals() та hashCode(). Поля, що беруть участь у цих методах, не повинні змінюватися після додавання до HashSet.

Які є переваги/недоліки використання HashSet у порівнянні з TreeSet?
Переваги HashSet: вища швидкодія, менші накладні витрати. Недоліки HashSet: відсутність впорядкованості елементів.

Чим поняття size відрізняється від capacity? Чи може size бути більший за capacity? Чи може capacity бути більший за size? Якщо так, то за яких умов?
Size - фактична кількість елементів у колекції. Capacity — кількість елементів, яку колекція може вмістити без розширення. Size не може бути більшим за capacity. Capacity може бути більшим за size після видалення елементів або після збільшення внутрішнього масиву.

Що таке load factor та на що і як він впливає?
Load factor - коефіцієнт заповнення хеш-таблиці. Він визначає момент, коли відбувається розширення та перехешування, і впливає на баланс між швидкодією та використанням пам’яті.

Що таке асоціативний масив? Чим ключ відрізняється від значення?
Асоціативний масив - структура даних, що зберігає пари ключ–значення. Ключ є унікальним ідентифікатором, значення є даними, пов’язаними з цим ключем.

Чи може в асоціативному масиві зберігатись кілька однакових ключів або однакових значень?
Однакові ключі зберігатися не можуть. Однакові значення можуть зберігатися.

До яких наслідків може привести зберігання неімутабельних об’єктів у HashSet, у HashMap в якості ключа та у HashMap в якості значення?
Можлива втрата доступу до елемента, некоректна робота contains() та remove(), порушення цілісності структури колекції. Найбільш критично для ключів у HashMap та елементів у HashSet.

Чи можуть об’єкти без інтерфейсу Comparable зберігатись у TreeMap?
Ні, якщо не задано Comparator. В іншому випадку виникає ClassCastException.

Чи можуть об’єкти без інтерфейсу Comparable зберігатись у HashMap? Чи взагалі впливає на щось наявність у об’єктів інтерфейсу Comparable при зберіганні у HashMap?
Так, можуть. Наявність інтерфейсу Comparable не впливає на роботу HashMap, оскільки вона використовує лише hashCode() та equals().
